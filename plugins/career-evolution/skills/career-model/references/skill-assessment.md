# Skill Assessment Rubric

## Level Definitions

| Level | Label | Definition |
|-------|-------|-----------|
| 1 | Novice | Have used it in tutorials/toy projects. Need to look up basic syntax. |
| 2 | Beginner | Can build simple features independently. Still reference docs frequently. |
| 3 | Intermediate | Productive in real projects. Understand patterns and trade-offs. Can debug most issues. |
| 4 | Advanced | Deep understanding. Can architect solutions, mentor others, handle complex edge cases. |
| 5 | Expert | Could teach a course or write a book. Know the internals. Go-to person in this area. |

## Trajectory Definitions

| Trajectory | Meaning |
|-----------|---------|
| Growing | Actively learning and practicing. Level expected to increase. |
| Stable | Maintaining current level. Using regularly but not deepening. |
| Declining | Not practicing. Level may decrease over time. |
| Pivoting | Deliberately shifting focus away from this skill. |

## Strategic Value Assessment

Rate 1-5 how valuable this skill is for each career path:
- **AI-Augmented Engineering Leader** (primary): How much does this skill help someone who builds AI tooling AND orchestrates AI workflows for teams?
- **Career Pivot** (watchlist): How transferable is this skill to adjacent fields?

Use the HIGHEST value across paths as the overall strategic value.

## When to Reassess

- After completing a significant project using the skill
- When a research session reveals market changes affecting the skill's value
- Every 3 months as part of regular review
- When trajectory changes (started or stopped practicing)
